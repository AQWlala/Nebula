# Memory Impact Assessment: add-agent-eval

> **change**: add-agent-eval
> **评估日期**: 2026-07-11
> **评估人**: AI

## 评估结论: ✅ 通过(评估结果不污染记忆层)

### 1. 对记忆层级的影响

| 层级 | 影响 | 说明 |
|------|------|------|
| L0 感官 | ❌ 无 | 评估器不产生感官输入 |
| L1 对话 | ❌ 无 | 评估器不参与对话流 |
| L2 经验 | ❌ 无 | 评估结果不写入 L2 |
| L3 事实 | ❌ 无 | 评估结果不写入 L3 |
| L4 语义 | ❌ 无 | 评估结果不做语义索引 |
| L5 反思 | ❌ 无 | 评估结果不写入 L5 |
| L6 蒸馏 | ❌ 无 | 评估结果不进入蒸馏 |
| L7 奇点 | ❌ 无 | 评估结果不影响核心价值 |

**关键决策**: 评估分数和评分卡只写入 `eval_runs` 表和 `EvolutionLog`,**不写入任何记忆层**。这避免了评估数据污染用户记忆上下文。

### 2. 对记忆容量的影响

| 项目 | 影响 | 量化 |
|------|------|------|
| SQLite 存储增长 | ⚠️ 可控 | eval_traces + eval_spans + eval_runs 表,单次评估约 10KB-100KB |
| Trace 收集开销 | ✅ 零(默认) | NEBULA_EVAL_TRACING=1 才启用,默认关闭 |
| 上下文 token 消耗 | ✅ 零 | 评估结果不注入上下文 |

### 3. 对记忆检索的影响

- 评估数据存储在独立的 SQLite 表中,与记忆表(`memories`)完全隔离
- 检索记忆时不会命中评估数据
- 不影响向量检索(LanceDB)的召回率

### 4. 对记忆压缩的影响

- 评估数据不进入 L3 事实层,不触发 BlackholeEngine 蒸馏
- 评估数据不占用 L3 容量配额
- EvolutionLog 有自己的归档策略(archive_rate_floor)

### 5. 对记忆同步的影响

- 评估数据**不参与跨设备同步**(由 sync/ 领域处理)
- 评估结果是设备本地的质量指标,不需要同步到其他设备
- 如果用户在多台设备上运行评估,各设备的评估结果独立

### 6. 对进化系统记忆的影响

- 评估分数作为 EvolutionEngine 的反馈信号,写入 EvolutionLog
- EvolutionLog 是进化系统自己的日志,不属于记忆层
- 评估结果不会因 EvolutionEngine 回滚而丢失(eval_runs 表独立)

## 风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| Trace 数据膨胀 | 中 | 低 | 定期清理 30 天前的 eval_traces |
| 评估结果误写入记忆 | 低 | 高 | 代码审查 + 测试验证评估数据不进 memories 表 |
| EvolutionLog 膨胀 | 中 | 低 | 现有 archive_rate_floor 机制处理 |
