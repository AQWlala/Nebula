# v2.3.0-macos-redesign — macOS 风格前端重设计

> **状态**: archived
> **领域**: ui
> **提出日期**: 2026-07-09
> **归档日期**: 2026-07-11
> **类型**: 大版本 (UI 重设计)

## 为什么

v2.2.0 的 Nebula 前端存在三个体验问题：

1. **导航混乱** — 10 个导航项平铺在侧边栏，无分组，用户难以快速定位。新用户反馈"找不到记忆入口在哪"。
2. **视觉沉重** — 纯色深蓝背景（`#0f1923` / `#152233`）在长时间使用后产生视觉疲劳，与 macOS 原生应用风格割裂，降低了"桌面工具"的定位感。
3. **状态栏浪费空间** — 底部独立状态栏独占一行（24px），仅显示模型/内存/版本三项信息，空间利用率低。

竞品调研（jiuwenswarm、openakita、hermes）表明，macOS 设计语言（毛玻璃侧边栏、大圆角卡片、Spotlight 搜索）能显著提升"桌面原生工具"的感知，且与 Nebula 的 Tauri 桌面定位天然契合。

## 做什么

1. 将侧边栏改为 macOS Finder 风格：毛玻璃背景、导航项按职能分 4 组（收藏/工作/监控/高级）
2. 状态栏融入侧边栏底部，释放底部空间
3. 新增顶部 44px macOS 风格 titlebar，含 Spotlight 风格搜索框（Cmd+K）
4. 调整色彩系统：从深蓝纯色改为中性灰 + 半透明分层（macOS 暗色模式启发）
5. 默认视图从 `code` 改为 `chat`（对话优先，降低上手门槛）
6. 内容区卡片采用大圆角（16px）+ 半透明背景

## 不做什么

- **不改后端** — 本变更纯前端，不触及任何 Rust 代码
- **不改路由逻辑** — ModeSwitcher 的模式判断逻辑不变，仅改默认值
- **不改 i18n 文案** — 导航分组标题复用现有 i18n key
- **不改移动端** — Nebula 当前仅桌面端，移动端适配留待后续
- **不引入新依赖** — 毛玻璃用 CSS `backdrop-filter`，无需额外库
