## 变更摘要
修复编译错误与 CI 问题:gRPC trait 名称冲突、hyper API 不匹配、reflect_test 死锁、CI 输出截断以及一批 clippy lint。

## 修复列表
1. gRPC trait 冲突(E0255)— `NebulaService` 重命名为 `ApiNebulaService`
2. hyper API 适配(E0061/E0277/E0308)— 使用 `TokioExecutor` / `TokioTimer` 适配 hyper 1.10+
3. reflect_test 死锁 — 持有 conn.lock() 期间调用 list_recent 触发重入,改用块作用域提前释放锁
4. CI 输出截断 — nextest exit 100 时 `| tee` 管道 buffer 丢失,改为直接重定向
5. clippy lint — `assert_eq!(x, true)` → `assert!(x)` 等一批修正
