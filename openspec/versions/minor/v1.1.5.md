# v1.1.5 — 启动修复(migrations 路径 / DB 路径 / 日志 / panic hook)

> **发布日期**: 2026-07-01
> **类型**: 小版本 (Patch version bump)
> **前一版本**: v1.1.4
> **变更范围**: migrations 路径, DB 相对路径, 日志默认写文件, panic hook

## 变更摘要
修复打包后无法启动的问题:migrations 路径在用户机器上不存在(改用内嵌 SQL)、DB 相对路径从快捷方式启动时解析失败、日志默认不写文件导致无法诊断、以及 panic 被静默吞掉。

## 修复列表
1. migrations 路径问题 — 改用 `include_str!` 将所有 SQL 文件内嵌到二进制中,新增 `run_bundled_migrations()` 和 `bundled_migration_status()`
2. DB 相对路径解析 — 在 setup 中将相对路径解析到 app data dir,修复从 System32 启动时 DB 创建失败
3. 日志默认写文件 — `init_tracing()` 默认写入平台 app data 目录(Windows: `%LOCALAPPDATA%\nebula\logs`)
4. panic hook — panic 信息写入 `nebula-panic.log` 文件,不再被 `windows_subsystem = "windows"` 静默吞掉
