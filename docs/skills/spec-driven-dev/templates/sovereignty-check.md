### 5. Checker 模型

**检查**: 本变更不会引入闭源 Checker 模型。

- [ ] ✅ 通过
- [ ] ❌ 违反
- [ ] ⚠️ 例外(需审批)

**说明**:

[说明本变更是否涉及 Checker 模型。若不涉及,确认。若涉及,确认使用本地 Ollama。]

> **红线**: Checker 处理代码 diff,发给云端 = "把你的代码变成别人的养料"。
> Checker 的 provider MUST 为 `ollama`(本地唯一允许)。

```
例(通过):
本变更不涉及 Checker 模型配置。
本变更的蒸馏 LLM 使用 Ollama(Maker 侧),与 Checker 无关。
若 Loop 使用 L4 蜂群模式审查本变更的代码,Checker 仍使用现有配置(provider=ollama)。
```

```
例(违反):
本变更新增了一个 Checker 配置项,允许指定 checker: claude-3-5-sonnet。
- 这违反了"Checker provider MUST 为 ollama"的红线
- 闭源 Checker 会将代码 diff 发送到 Anthropic 服务器
修复方案: 删除该配置项,Checker provider 硬编码为 ollama,不可配置。
```
