## 记忆系统行为变更

### 检索行为变更

[变更后,记忆检索行为有何不同?]

```
例:
检索优先级变更:
  之前: L4 > L3 > L2 > L1 > L0
  之后: L4 > L6 > L3(未蒸馏) > L3(已蒸馏) > L2 > L1 > L0

L6 胶囊优先于已蒸馏的 L3 事实返回(胶囊更精炼)。
未蒸馏的 L3 事实仍优先返回(信息更完整)。
```

### 存储行为变更

[变更后,记忆存储行为有何不同?]

```
例:
- 新增 l6_capsules 表
- memory_facts 表新增 is_distilled + distilled_at 字段
- 蒸馏后 L3 事实保留但标记,不删除
- L6 胶囊独立存储,有自己的 embedding 索引
```

### 遗忘策略变更

[变更后,遗忘策略有何不同?]

```
例:
- 之前: L3 事实按 LRU 淘汰(容量超限时删除最旧的)
- 之后: L3 事实先蒸馏为 L6,再按 LRU 淘汰(保留语义信息)
- 已蒸馏的 L3 事实优先被淘汰(语义已在 L6 保留)
```

## 回滚策略

<!-- 若变更导致记忆系统退化,如何回滚? -->

[描述回滚方案。]

```
例:
回滚步骤:
1. 停止蒸馏触发器(禁用阈值触发 + 取消定时任务)
2. l6_capsules 表数据可保留(不影响现有功能)
3. memory_facts.is_distilled 字段可保留(默认 false,不影响检索)
4. 若需完全回滚:
   a. DROP TABLE l6_capsules
   b. ALTER TABLE memory_facts DROP COLUMN is_distilled
   c. ALTER TABLE memory_facts DROP COLUMN distilled_at
   d. 回滚代码到上一个版本

回滚安全性:
- 蒸馏是纯新增操作,不修改/删除现有 L3 事实内容
- 回滚不丢失任何记忆数据(L3 事实未被删除)
- L6 胶囊可安全丢弃(是 L3 的衍生物)
```
