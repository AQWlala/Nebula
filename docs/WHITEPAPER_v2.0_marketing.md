## 1. 愿景与使命

### 1.1 一句话愿景

**让 AI 成为大脑的延伸，而不是大脑的替代。**

### 1.2 我们看到的未来

当前 AI 行业正朝"更大、更强、更云端"的方向狂奔。模型越来越聪明，但用户的思考越来越被托管——你跟 ChatGPT 聊过的每一句话、Notion AI 整理过的每一篇笔记，都存在别人的服务器上。一旦服务停服、提价、改隐私协议，多年积累瞬间归零。

我们相信另一条路：**让 AI 回到本地，让记忆回到你的手中，让思考回到你的主权。**

### 1.3 使命

Nebula 的使命是打造一款**你真正拥有的 AI 第二大脑**——会记住、会思考、会进化，但**永远属于你**。

- **会记住**：5 层记忆架构，让 AI 不再"失忆"
- **会思考**：6 Agent 蜂群协作，处理复杂任务
- **会进化**：L5 元认知反思，自我改进
- **属于你**：本地存储 + E2EE 同步 + MIT 开源

### 1.4 品牌原型

Nebula 的品牌原型是 **魔法师（Magician）+ 智者（Sage）**。

- 魔法师的承诺是"转化与生长"——把零散对话炼成有生有灭、自我演化的星云
- 智者赋予它"懂你、复盘你"的灵魂——L5 反思让它像真正的智者一样审视自己

星云的隐喻贯穿整个产品：记忆有生有灭（黑洞引擎压缩、海绵引擎吸收），知识自我演化（L4 抽象、L5 反思），浩瀚无垠（LLM Wiki 双向链接织成星图）。

## 2. 问题陈述：我们到底在解决什么

### 2.1 这个时代的知识困境

> **你今天读过的文章、写过的代码、聊过的灵感——明天还记得多少？**

浏览器历史会清空，聊天记录会沉底，笔记软件越用越乱。我们都在生产知识，却没人真正拥有它。

### 2.2 用户的五大深层痛点

| 痛点 | 描述 | 情绪强度 |
|------|------|---------|
| **遗忘焦虑** | 三个月前的灵感像从未发生过——知识工作者最大的恐惧是"我的思考在流失" | 🔴 极强 |
| **隐私让渡的不安** | 用 ChatGPT 越顺手，越觉得把大脑租给了别人。数据主权焦虑挥之不去 | 🔴 极强 |
| **AI"失忆"的挫败感** | 每次对话都要重新解释项目背景、写作风格、代码偏好。AI 听起来很聪明，却记不住"你是谁" | 🟠 强 |
| **笔记越用越乱** | Notion/Obsidian 越堆越像垃圾场，标签没人维护，知识库变成"再也打不开的抽屉" | 🟠 强 |
| **AI 成本失控** | 烧 Token 像烧钱，复杂任务一跑几十刀，越用越肉疼，最终"不敢用" | 🟡 中强 |

### 2.3 现有方案的局限

| 方案 | 局限 |
|------|------|
| ChatGPT / Claude | 数据归属厂商，无长期记忆，月费 $20+，停服即归零 |
| Notion AI | 云端存储，无 E2EE，AI 只是文档附庸，不是记忆系统 |
| MemGPT | 仅 3 层记忆，无元认知反思，无 GUI，无蜂群协作 |
| Supermemory | 云端 SaaS，简单检索，无深度记忆架构 |
| Obsidian + AI 插件 | 本地但 AI 能力拼装，无原生记忆系统，无多 Agent |
| Apple Intelligence | 端云混合，系统级浅记忆，绑定硬件，不可迁移 |

**市场空白**：没有一款产品同时满足"本地优先 + 深度记忆 + 多 Agent + E2EE + 开源"。

## 3. 解决方案：Nebula 是什么

### 3.1 一句话定位

**Nebula 是一款本地优先的 AI 第二大脑，通过 5 层演化记忆与蜂群协作，让 AI 替你记住、与你共同思考。**

### 3.2 三个关键判断

1. **不是云端 SaaS，不是套壳 API**——基于 Rust + Tauri 2.0 的桌面应用，数据默认本地存储
2. **核心差异不在"模型更强"，而在"记忆更深"**——L0 缓存到 L5 元认知反思的分层架构
3. **把"个人 AI 主权"做成可负担的工程现实**——E2EE 同步、Ollama 本地推理、SemanticCache 降 70% Token

### 3.3 不是又一个 ChatGPT 套壳

Nebula 的真正野心不是"聊天"，而是 **记忆**：

- 你和它聊过的每一句话，它会**记住**
- 你写过的代码、调研过的资料，它会**沉淀**
- 你三个月前的灵感，它会**主动回忆**
- 它会**反思**自己的判断，**进化**自己的能力
- 所有这一切，**数据只属于你**

## 4. 核心能力：四个让 Nebula 不同的理由

### 4.1 🧠 5 层记忆架构 — 不是聊天记录，是会演化的记忆

业界大部分"AI 记忆"只是把历史对话塞回 Prompt。Nebula 不一样：

| 层级 | 名称 | 角色 | 状态 |
|------|------|------|------|
| **L0** | 缓存 | 当前会话上下文，毫秒级响应（256 条 LRU + SessionWindow） | ✅ |
| **L1** | 消息 | 原始对话/操作流水，7 天保留 | ✅ |
| **L2** | 经验 | 实体关联、概念网络 | ✅ |
| **L3** | 事实 | 结构化知识 + 技能库 | ✅ |
| **L4** | 知识 | 跨任务抽象 + 用户偏好 + 价值对齐（ConstitutionalAI） | ✅ |
| **L5** | 教训 | **元认知反思，自我改进** ⚡ | ✅ |

**两个独特引擎让记忆有生有灭**：

- **黑洞引擎（BlackholeEngine）**：低重要性记忆自动"致密化"归档，永不删除，可追溯
- **海绵引擎（SpongeEngine）**：高价值记忆主动吸收，cosine > 0.85 阈值去重合并

**你的记忆像真正的星云——有生有灭，自我演化。**

### 4.2 🐝 蜂群协作 — 一个 Agent 不够，那就六个

复杂任务从来不是一个人能搞定的。Nebula 内置 6 种专业 Agent：

| Agent | 角色 | 典型场景 |
|-------|------|---------|
| 🎨 **Writer** | 撰写文档、博客、提案 | 技术博客、API 文档、发布说明 |
| 💻 **Coder** | 实现代码、修复 Bug | 代码实现、PR 审查 |
| 🔍 **Reviewer** | 审查、校对、挑刺 | 事实核查、一致性检查 |
| 📚 **Researcher** | 调研背景、整理资料 | 主题调研、不确定性标注 |
| 📋 **Planner** | 拆解任务、分配调度 | 任务分解、子任务分配 |
| 🛠️ **Generic** | 通用兜底 | 任意场景 |

**协作机制**：
- **AgentBus** 消息总线：点对点 + 广播通信
- **Negotiator** 协商：置信度投票 + LLM 仲裁 + 降级策略
- **MasterOrchestrator**：基于 petgraph 的 DAG 任务图，拓扑分层执行
- **LeaderElector**：动态选举主 Agent
- **ToT 思维树**：复杂决策的多路径探索

这不是简单的"多 Agent"，是真正的**协作工作流**。

### 4.3 🔐 隐私优先 — 你的记忆，只属于你

| 安全能力 | 实现 |
|---------|------|
| **本地存储** | SQLite + LanceDB，数据默认不上云 |
| **E2EE 同步** | X25519 密钥交换 + HKDF-SHA256 + AES-256-GCM，私钥永不出设备 |
| **SSRF 防护** | 拦截对内网的请求 |
| **Prompt 注入检测** | 扫描恶意输入 |
| **Shell 白名单** | 仅允许预授权命令 |
| **KeyVault** | OS Keychain 优先，AES-256-GCM 文件降级 |
| **DB 加密** | SQLCipher 全库加密 |

**你不用把记忆上传到别人的服务器，也能跨设备同步。**

### 4.4 💰 成本可控 — 不烧钱的 AI 才能用得起

| 优化手段 | 效果 |
|---------|------|
| **SemanticCache** | 语义缓存，cosine ≥ 0.92 短路返回，**降 70% Token** |
| **TokenJuice** | 三级压缩，减少输入 Token |
| **ModelRouter** | 智能路由到最优模型（本地优先） |
| **日预算** | 超限自动切换免费本地模型 |
| **本地推理** | Ollama 跑 qwen2.5/deepseek，零 API 费用 |
| **Arena** | ELO 评分 A/B 测试不同模型，持续优化 |

## 5. 目标用户：谁在用 Nebula

### 5.1 三个核心用户画像

#### Persona 1：隐私敏感的知识工作者（研究员/咨询顾问）

- **痛点**：笔记越积越乱、跨设备同步泄露风险、三个月前的调研找不回
- **动机**：拥有可审计、可加密、可演化知识网络
- **Nebula 价值**：E2EE 同步 + LLM Wiki 双向链接 + L2/L4 概念沉淀，把零散调研织成星图，且私钥不出设备
- **典型场景**："三年前我写过一篇关于分布式锁的文章，Nebula 不仅记得，还能在新文章里自动引用我当时的论证逻辑。"

#### Persona 2：长期写作者/独立开发者

- **痛点**：ChatGPT 不记得你半年前的代码风格与论证逻辑，每次重启对话即失忆
- **动机**：让 AI 形成"个人化长期语境"，降低重复解释成本
- **Nebula 价值**：L4 偏好层 + 版本化记忆 + 蜂群 Writer/Reviewer，能在新作品中主动引用历史风格与判断
- **典型场景**："我对 Reviewer Agent 说'检查这个 PR'，它直接引用了我半年前的代码评审标准。它真的'懂'我。"

#### Persona 3：极客/技术早 adopter

- **痛点**：厌恶订阅费、想跑本地模型、要完全掌控数据
- **动机**：MIT 开源 + Ollama + 可扩展 Skill/MCP 生态
- **Nebula 价值**：零月费、SQLCipher 全库加密、257 个 Tauri 命令深度可玩、Sidecar 进程架构可演进
- **典型场景**："我把每天的灵感随手丢给 Nebula，它自动抽取实体、关联概念。半年后回看，它已经替我织出了一张知识星图。"

### 5.2 三个 Aha Moment

| Aha Moment | 触发场景 | 情感反馈 |
|------------|---------|---------|
| **第一次"被回忆"** | 写新文章时，Nebula 主动引用你半年前的论证逻辑 | "它真的记得我" |
| **第一次看蜂群吵架** | Reviewer Agent 挑刺 Coder Agent 的代码，APPROVE/REVISE/REJECT 流转 | "这是真正的协作" |
| **第一次零成本长跑** | 跑完复杂多 Agent 任务，发现没花一分 API 费 | "AI 我也用得起" |

### 5.3 三个留存钩子

1. **记忆资产化**：用得越久，L2-L5 记忆越厚，迁移成本越高——你的"AI 人格"无法搬走
2. **反思闭环**：L5 元认知定期反思，让用户感到"它在替我成长"，形成每日回看的仪式感
3. **场景模板依赖**：28 个场景模板（周报、站会、OKR、PR 审查…）嵌入工作流，成为肌肉记忆

## 7. 商业模式：可持续的演化

### 7.1 商业价值锚点

Nebula 的商业价值不在"AI"，而在 **"对 AI 记忆的主权"**。

### 7.2 四条变现路径

| 路径 | 定价模型 | 目标用户 | 难度 |
|------|---------|---------|------|
| **云同步与中继服务** | $5/月 或 $48/年 | 多设备知识工作者 | 中（E2EE Relay 已就绪） |
| **企业版授权（BSL/双协议）** | $20/用户/月 + 私有部署 $5K/年 | 律所、金融机构、研发团队 | 高（需销售团队） |
| **Skill Marketplace 分成** | 30% 抽成 + 官方精品 Skill $3-$10 | 高级开发者、效率玩家 | 中（v2.1 已规划 Hub） |
| **托管推理网关** | 按 Token 计费，比官方便宜 20% | 不愿配 Ollama 的小白用户 | 低（SemanticCache 已降 70% 成本） |

### 7.3 三个增长飞轮

**飞轮 A · 隐私口碑飞轮**
数据主权焦虑 → 用户因 E2EE+本地存储选择 Nebula → 社媒输出"我的 AI 记忆属于我" → 吸引同好 → Star/DAU 上升 → 贡献者增加 → 安全审计更厚 → 信任强化

**飞轮 B · 记忆复利飞轮**
首日录入 10 条知识 → L2/L3 自动建联 → 半年后主动回忆带来"惊艳时刻" → 用户主动晒图 → 新用户冷启动即感受复利 → 留存率上升 → 记忆池更深 → 召回质量更高

**飞轮 C · Skill 生态飞轮**
Skill Marketplace 上线 → 开发者发布垂直 Skill → 用户付费 → 开发者分成 → 更多 Skill → 平台价值上升 → 跨过网络效应临界点

### 7.4 MIT 协议下的商业策略

采用 **Open Core** 模式：
- **核心开源（MIT）**：记忆 / 蜂群 / E2EE / LLM Wiki / 本地推理
- **企业闭源**：SSO、审计日志、主从同步、合规导出
- **商标保护**：注册"Nebula"商标，防止云厂商 fork 白嫖
- **不可 fork 资产**：Skill Marketplace 网络效应 + 用户记忆复利

## 9. 社区与生态：Nebula 属于所有人

### 9.1 开源哲学

Nebula 是 MIT 开源项目。我们相信：
- **AI 应该是大脑的延伸，不是大脑的替代**
- **数据主权不可让渡**
- **开源是建立信任的唯一方式**

### 9.2 三个月传播路线图

| 月份 | 主题 | 渠道 | 内容形式 |
|------|------|------|---------|
| **M1** | "AI 第二大脑宣言" — 隐私主权叙事 | HN、V2EX、X、少数派 | 宣言长文 + 60 秒演示 GIF + 对比表 |
| **M2** | "技术深潜" — 5 层记忆 + 蜂群架构拆解 | B 站、YouTube、知乎专栏 | 30 分钟架构视频 + 白皮书摘要 |
| **M3** | "社区启航" — v2.1 路线图 + 贡献者征集 | GitHub Discussions、Discord、即刻 | 路线图直播 + good-first-issue 攻略 |

### 9.3 贡献者生态

降低贡献门槛的三项措施：
- 开通 Discord 社区（#help、#good-first-issue 频道）
- 5 分钟 dev 环境一键脚本（含 docker compose ollama）
- "PR Buddy"机制：每个新贡献者配一名维护者结对 review

**Good First Issue 清单**：
1. `[i18n]` 补充日语/韩语翻译
2. `[docs]` 为 6 个 Agent 各写一篇使用案例
3. `[ui]` 给 DAG Canvas 添加节点右键菜单
4. `[test]` 为 SemanticCache 补充 property-based test
5. `[chore]` 把 36 个 SQL 迁移生成可视化 changelog
